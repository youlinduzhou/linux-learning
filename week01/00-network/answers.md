## 分析层（L4）

4. 对比 traceroute 加 -n 和不加 -n 的差异：
   （你实测了43秒 vs 17秒，分析一下为什么差这么多）
因为加-n 不解析主机名，直接显示数字；不加-n 要追踪数据包经过的路由，每一跳都要做 DNS 反向解析（IP→主机名）。
traceroute baidu.com        ← 每一跳都要做 DNS 反向解析（IP→主机名），很慢
traceroute -n baidu.com     ← 直接显示 IP 数字，不做域名解析，很快

5. 对比 0.0.0.0 和 127.0.0.1 绑定端口的区别：
   （结合今天的对话和你的运维经验来写）
 0.0.0.0（通配/所有接口）,通过任何一个IP地址都能访问这个端口，相当于：把大门和窗户全打开，任何入口都能进来。
127.0.0.1（本地回环地址），只有本机自己能访问，相当于：你家的内线电话，只有屋子里的人能打。

## 评价层（L5）

6. 如果你在VM中 ping baidu.com 出现 100% packet loss，你会按什么顺序排查？
   （设计一个排查流程，可以参考你之前说的物理层逐段检查思路）
如果出现 100% packet loss，说明测试包全丢失，可从以下四方面排查：
第1步：ping 127.0.0.1        → 看本机协议栈是否正常
  如果通 → 第2步
  如果不通 → 系统网络层问题（重装/检查内核）

第2步：ping 网关192.168.5.1  → 看局域网是否正常
  如果通 → 第3步
  如果不通 → 物理链路/路由器问题

第3步：ping baidu.com        → 看外网是否正常
  如果不通 → DNS问题/防火墙/运营商问题

7. 你觉得Day0的5个网络命令中，哪个对你未来的DevOps工作最有价值？为什么？
我认为在 DevOps 日常中，ip 命令最有价值。
因为：它能直接查看并修改网络配置（地址、路由等），是排错与变更的基础，而其他命令大多只能诊断或查看状态，无法动手修复。
