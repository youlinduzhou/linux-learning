## 记忆层（L1）

1. 默写Linux核心目录及其用途（至少8个）：
   /etc = "系统配置中心"，存放系统设置和类似于Windows注册表的文件
   /var = 存放经常变化的文件，日志、数据库、网站文件,例如www、snap、opt、log、cache文件夹
   /home = 普通用户家目录，root管理员的家目录是 /root
   /bin = 存放所有用户都能用到的可执行程序
   /lib = 存放程序运行所需的库文件
   /share = 存放图标、主题、文档等独立于硬件架构的数据
   /tmp = 存储系统运行时的一些缓存文件，类似于Windows系统的temp文件夹
   /dev = 能看见一切硬件，几乎所有的硬件都能在/dev下找到，更像是一个设备调度中心
   /proc = 查看内核与硬件信息、查看和调整进程信息,例如查看当前的内核版本：cat /proc/version

2. ls命令显示所有文件（含隐藏文件）的参数：ls -la

## 理解层（L2）

3. 从/home/k8s-master执行 cd ../../etc，最终会进入哪个目录？为什么？
   最终会进入/etc文件夹，因为..是每次只返回一级，用了两次../..就是返回了两级，然后进入了etc文件夹目录

4. 用一句话解释 ~ - . .. 四个特殊符号的含义：
     ~ = 当前用户家目录的意思，例如普通用户输入cd ~ 按下确认键后就进入到该普通用户的家目录了，如果输入sudo -i切换到管理员了，那输入cd ~就是返回管理员的家目录
     - = 在 路径上下文 中， - 代表 上一个工作目录  例如: cd -
     . = .根据情况不同意思也不同，如果在文件的前面那表示隐藏文件，例如.cache，如果是在路径中，例如：./home = 当前目录下的 home
     .. = 每次只返回一级，如果用了两次../..就是返回了两级

## 分析层（L4）

7. Linux文件系统（单根/）和Windows（多盘符C: D:）的设计各有什么优缺点？
   Linux文件系统（单根/）
       优点：统一命名空间，脚本可移植性强；逻辑抽象让硬件细节透明。 缺点：新手难以理解设备挂载点。 DevOps影响：Docker/K8s天然适配Linux文件系统。       
   Windows多盘符  
      优点：物理隔离直观，单盘符损坏不影响其他盘。 缺点：脚本必须硬编码盘符（C:\ D:\），跨机器易出错。  DevOps影响：Windows容器需要额外映射。

8. 浏览/var目录的内容，你看到哪些子目录？它们各自存放什么类型的数据？
   log/ → 系统日志（最重要！）
   www/ → Web服务器根目录（Nginx/Apache）
   lib/ → 包管理器数据库
   cache/ → 应用程序缓存
   tmp/ → 临时文件
   backups/ → 备份文件
   snap/ → Snap包管理器数据
   
## 评价层（L5）

9. 有人建议把所有文件都放在/home下方便管理，不使用其他系统目录。你如何评价这个做法？
   1.安全风险 ：系统关键目录（/etc, /bin）有专门权限保护，/home是777或755，安全性差
   2.运维混乱 ：日志、配置、可执行程序混在一起，排查问题困难
   3.违反FHS标准 ：Linux Filesystem Hierarchy Standard规定各目录职责
   4.自动化工具依赖 ：Docker、Ansible等工具默认查找标准路径
   5.备份策略复杂 ：无法针对不同类型数据制定差异化备份方案
