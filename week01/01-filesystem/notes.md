## 一、核心知识点总结

### 1.1 Linux目录结构（FHS标准）

| 目录 |                                    用途                                    | Windows类比 |
|------|---------------------------------------------------------------------------|------------|
| /etc | /etc 的主要用途是系统配置中心，存放系统设置，如果用Windows的概念来类比，它最像注册表 | 类似注册表 |
| /var | 存放经常变化的文件，日志、数据库、网站文件                                       | 临时+日志 |
| /home | 普通用户家目录，root管理员的家目录是 /root                                    | C:\Users\ |
| /usr | 用户程序和数据                                                              | Program Files |
| /bin | 存放所有用户都能用到的可执行程序                                               | System32 |
| /tmp | _存储系统运行时的一些缓存文件                                                  | Temp |
| /dev | 能看见一切硬件，几乎所有的硬件都能在/dev下找到，更像是一个设备调度中心              | 设备管理器 |
| /proc | 查看内核与硬件信息、查看和调整进程信息,例如查看当前的内核版本：cat /proc/version   | 任务管理器 |

### 1.2 关键命令速查

#### 基础导航命令
```bash
pwd          # 显示当前工作目录
ls -la       # 列出所有文件（含隐藏文件）
cd <path>    # 切换目录
cd -         # 返回上层目录
```

#### 特殊路径符号
```bash
~            # 当前用户家目录
.            # 当前目录
..           # 上级目录
-            # 上一个工作目录
```

#### 目录操作命令
```bash
mkdir -p     # 递归创建目录
tree -L N    # 查看N层目录结构
file <file>  # 查看文件类型
```

## 二、重点概念理解

### 2.1 绝对路径 vs 相对路径

**绝对路径**：从根目录 `/` 开始的完整路径
- 示例：`/home/k8s-master/linux-notes`
- 优点：不受当前工作目录影响，永远指向同一位置
- 适用场景：脚本编写、系统配置（如 PATH 环境变量）

**相对路径**：从当前位置开始的路径
- 示例：`../../week01/01-filesystem`
- 优点：书写简短，方便批量移动或复制目录结构
- 风险：依赖当前所在位置，换一个目录执行就会出错或指向错误文件

### 2.2 文件类型识别

通过 `ls -la` 第一列字符判断：
```
d rwxr-xr-x   → d = 目录（directory）
- rwxr-xr-x   → - = 普通文件
l rwxr-xr-x   → l = 符号链接（link）
```

### 2.3 隐藏文件规则

- 以 `.` 开头的文件为隐藏文件
- 常见隐藏文件用途：
  - `.bashrc` → 交互式非登录 Shell 的初始化脚本，用于定义别名、函数、PATH 等个性化配置
  - `.ssh/` → 存放 SSH 协议的配置文件和密钥对
  - `.cache/` → 符合 XDG 规范的用户缓存目录，存放可随时重建的非必要数据，删除后不影响程序正常运行

## 三、易错点记录

### ❌ 错误1：/home 与 /root 混淆
**错误理解**：/home 是所有用户的家目录（包括root）
**正确理解**：
- `/home/普通用户名` → 普通用户家目录
- `/root` → root管理员专属家目录（不在/home下）

**验证方法**：
```bash
whoami          # 查看当前用户
pwd             # 查看当前位置
sudo -i         # 切换到root
pwd             # 应该显示 /root
```

### ❌ 错误2：mkdir 花括号展开语法
**错误命令**：`mkdir -p ~/{test,a,b}` → 创建3个平行目录
**正确命令**：`mkdir -p ~/test/a/b/c` → 创建4层嵌套目录

**记忆口诀**：花括号 `{ }` 是"并行"，斜杠 `/` 是"层级"

### ❌ 错误3：tree命令参数格式
**错误写法**：`tree L 2 /etc`
**正确写法**：`tree -L 2 /etc`

**原因**：Linux命令参数需要 `-` 前缀

## 四、DevOps实战应用场景

### 4.1 日志分析时的路径选择
```bash
# 推荐使用绝对路径（脚本中）
tail -100 /var/log/syslog

# 日常操作可用相对路径（注意当前位置）
cd /var/log && tail -100 syslog
```

### 4.2 自动化部署时的目录结构设计
```bash
mkdir -p ~/devops-learning/{01-filesystem,02-text-processing,scripts,configs,logs,notes}
```

### 4.3 Docker/K8s与Linux文件系统的关系
- Docker镜像基于Linux文件系统分层
- K8s Volume挂载依赖绝对路径
- 容器内路径映射遵循FHS标准

### ⚠️ 需要加强练习的内容
1.  tree命令参数格式（需形成肌肉记忆）
2.  tree -L 2 /etc    # 查看/etc目录前2层结构
3.  tree -a -s -h     # 显示所有文件（包括隐藏）及人类可读的大小
4.  source commands.sh 与 bash commands.sh 命令的核心区别：
source commands.sh ：
    在当前 shell 进程中执行，会影响当前 shell（变量、函数、工作目录等修改都会保留），加载配置文件（.bashrc, .profile）、定义环境变量、切换目录，不要求脚本有可执行权限。
bash commands.sh ：
    启动一个新的子 shell 执行，不影响当前 shell（脚本里的一切修改，脚本结束后就消失），运行独立的脚本程序，不希望它污染当前环境，不要求可执行权限（bash 只是解释器）
